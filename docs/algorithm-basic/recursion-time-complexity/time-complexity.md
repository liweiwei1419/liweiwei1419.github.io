「时间复杂度」用于评估算法的好坏。它是一个渐进概念，需要用 **动态** 的观点去理解。

本节内容讲解了时间复杂度的严格定义（极限形式），以便大家理解时间复杂度的计算规则。并且还指出了：时间复杂度不是程序的运行时间，应该使用「空间换时间」，更多关注在优化「时间复杂度」。
